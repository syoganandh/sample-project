Object Oriented Programming (OOPs)

OOP models real-world entities as objects with data (variables) and behavior (methods). Java supports all OOP pillars: Class, Object, Encapsulation, Abstraction, Inheritance, Polymorphism, and Message Passing.

🏛️

Class

Blueprint/model. Virtual encapsulation of properties and behaviors. Generalization.

🎯

Object

Physical instance of a class. Real entity with actual state. Specialization.

🔒

Encapsulation

Binding data + code together. Use private fields with public getters/setters.

🎭

Abstraction

Show necessary details, hide implementation. Abstract classes & interfaces.

🧬

Inheritance

Getting variables/methods from parent class via extends. Code Reusability.

🔀

Polymorphism

"Many forms" — overloading (compile-time) & overriding (runtime). Flexibility.

📨

Message Passing

Transferring data between objects/instructions. Improves communication.

// CLASS — blueprint for Employee objects
class Employee {
    // Instance Variables (data/properties)
    String eid   = "E-111";
    String ename = "Durga";
    float  esal  = 25000.0f;

    // Method (behavior)
    public void display() {
        System.out.println("ID: "   + eid);
        System.out.println("Name: " + ename);
        System.out.println("Sal: "  + esal);
    }
}

class Test {
    public static void main(String[] args) {
        // OBJECT — real instance of Employee
        Employee emp = new Employee();
        emp.display();
    }
}

// ENCAPSULATION — binding data + code; hiding internal state
class BankAccount {
    // 🔒 Private fields — not directly accessible
    private String owner;
    private double balance;

    public BankAccount(String owner, double bal) {
        this.owner   = owner;
        this.balance = bal;
    }

    // 🔓 Getter — controlled read
    public double getBalance() { return balance; }

    // ✏️ Setter — with validation
    public void deposit(double amt) {
        if(amt > 0) balance += amt;
    }
}

// acc.balance = -9999;  ❌ Compile Error — private!
BankAccount acc = new BankAccount("Alice", 500.0);
acc.deposit(200.0);
System.out.println(acc.getBalance()); // 700.0
// INHERITANCE — getting variables/methods from parent
// Goal: CODE REUSABILITY
class Employee {               // Parent / Super class
    String eid, ename, eaddr;
    void getEmpDetails() {
        System.out.println(ename);
    }
}

class Manager extends Employee {   // Child class
    String department;
    // Reuses eid, ename, eaddr, getEmpDetails()
}

class Engineer extends Employee {  // Another child
    String techStack;
}

// Types of Inheritance in Java:
// ✓ Single    → A extends B
// ✓ Multilevel → A extends B extends C
// ✓ Hierarchical → B extends A, C extends A
// ❌ Multiple → NOT supported (use interfaces)
// POLYMORPHISM — "many forms"

// 1. Method Overloading (Compile-time Polymorphism)
class Calculator {
    int    add(int a, int b)       { return a + b; }
    double add(double a, double b) { return a + b; }
    int    add(int a, int b, int c){ return a+b+c; }
}

// 2. Method Overriding (Runtime Polymorphism)
class Animal {
    void sound() { System.out.println("Generic sound"); }
}
class Dog extends Animal {
    @Override
    void sound() { System.out.println("Woof!"); }
}
class Cat extends Animal {
    @Override
    void sound() { System.out.println("Meow!"); }
}

// Dynamic dispatch
Animal a = new Dog();
a.sound(); // → Woof! (decided at runtime)
Woof!
// ABSTRACTION — hide implementation, show interface

// 1. Abstract Class
abstract class Shape {
    abstract double area();     // no body — must override
    void describe() {           // concrete method
        System.out.println("I am a shape");
    }
}
class Circle extends Shape {
    double r;
    Circle(double r) { this.r = r; }
    double area() { return 3.14 * r * r; }
}

// 2. Interface (100% abstraction)
interface Printable {
    void print();   // implicitly public abstract
}
class Doc implements Printable {
    public void print() {
        System.out.println("Printing document");
    }
}
💡OOP vs OB Languages: OOP languages (Java) support ALL features including Inheritance. Object-Based languages (JavaScript) support all features except Inheritance.
