Multi Threading

A thread is a lightweight sub-process — the smallest unit of execution. Multi-threading allows multiple threads to run in parallel within a single program, reducing execution time and improving performance.

🏋️

Process

Heavyweight — has its own memory space. Context switching is slow and costly.

🪶

Thread

Lightweight — shares process memory. Context switching is fast, improves performance.

🔄

Multi-tasking

Multiple tasks load into memory at once. Multiple processes/threads execute in parallel.

🔐

Synchronization

Ensures only one thread accesses a resource at a time — prevents race conditions.

New — Thread object created, not started yet.
Runnable — start() called, waiting for CPU time.
Running — Thread Scheduler assigns CPU, run() executes.
Blocked/Waiting — waiting for I/O, lock, or sleep().
Dead — run() completes or stop() is called.

// Method 1: Extend Thread class
class MyThread extends Thread {
    @Override
    public void run() {
        for(int i = 1; i <= 3; i++) {
            System.out.println(getName() + " → " + i);
        }
    }
}

class Test {
    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        MyThread t2 = new MyThread();
        t1.start();  // ✅ start() — creates new thread
        t2.start();  // ❌ Not run() — run() uses current thread
    }
}

// Method 2: Implement Runnable (preferred)
// Advantage: class can still extend another class
class MyTask implements Runnable {
    @Override
    public void run() {
        for(int i = 1; i <= 3; i++) {
            System.out.println(
                Thread.currentThread().getName() + " → " + i
            );
        }
    }
}

class Test {
    public static void main(String[] args) {
        MyTask task = new MyTask();
        Thread t1 = new Thread(task, "Worker-1");
        Thread t2 = new Thread(task, "Worker-2");
        t1.start();
        t2.start();
    }
}
// Method 3: Lambda Expression (Java 8+) — shortest
Thread t1 = new Thread(() -> {
    for(int i = 1; i <= 3; i++)
        System.out.println("T1 → " + i);
});

Thread t2 = new Thread(() ->
    System.out.println("T2 running")
);

t1.start();
t2.start();

// Daemon thread — background service thread
Thread daemon = new Thread(() -> { /* task */ });
daemon.setDaemon(true);  // dies when main thread ends
daemon.start();
// Problem: Race condition without synchronization
class Counter {
    private int count = 0;

    // synchronized — only one thread at a time
    public synchronized void increment() {
        count++;
    }

    public int getCount() { return count; }
}

// Synchronized block — finer control
public void transfer(double amt) {
    synchronized(this) {    // lock on this object
        balance -= amt;
    }
}

// Context switching types:
// Heavy Weight → between Processes (slow)
// Light Weight → between Threads (fast)
Thread t = new Thread(task);

t.start()                // start thread execution
t.getName()              // get thread name
t.setName("MyThread")   // set thread name
t.getPriority()          // 1(MIN) to 10(MAX), default 5
t.setPriority(8)         // set priority
t.isAlive()              // true if running
t.isDaemon()             // true if daemon thread
t.join()                 // wait for t to finish
t.interrupt()            // interrupt sleeping thread

Thread.sleep(1000)       // sleep 1 second (static)
Thread.currentThread()   // reference to running thread
Thread.yield()           // hint to scheduler to pause
💡Always call start() — not run(). Calling run() directly executes in the current thread as a normal method call, not in a new thread.
