Java Packages

A package is a folder containing related .class files representing related classes and interfaces as a single unit. Packages provide Modularity, Abstraction, Security, Sharability, and Reusability.

🧩

Modularity

Divide large code into smaller, manageable units

🎭

Abstraction

Classes hidden from outside by default

🔐

Security

Encapsulation + Abstraction = better security

🔗

Sharability

Declare once, share across multiple apps

♻️

Reusability

Import and reuse in any number of projects

// Contains: String, System, Object, Thread, Math...

// Step 1: Declare the package at top of file
// File: D:\javaapps\com\myapp\models\Student.java

package com.myapp.models;    // must be first statement

public class Student {
    private int    sid;
    private String sname;

    public Student(int id, String name) {
        this.sid   = id;
        this.sname = name;
    }

    public void display() {
        System.out.println(sid + " - " + sname);
    }
}

// Step 2: Use in another class
package com.myapp;
import com.myapp.models.Student;

public class Main {
    public static void main(String[] args) {
        Student s = new Student(101, "Durga");
        s.display();
    }
}
// 1. Import specific class (recommended)
import java.util.ArrayList;
import java.util.HashMap;

// 2. Import entire package using wildcard *
import java.util.*;   // all classes in java.util

// 3. Static import (Java 5+) — for static members
import static java.lang.Math.PI;
import static java.lang.Math.*;
System.out.println(PI);         // no Math. prefix needed
System.out.println(sqrt(16));    // 4.0

// 4. Fully Qualified Name (no import needed)
java.util.ArrayList<String> list = new java.util.ArrayList<>();
// Compile with -d flag to create package directories
// D:\javaapps> javac -d . Student.java
// Creates: D:\javaapps\com\myapp\models\Student.class

// Run using fully qualified class name
// D:\javaapps> java com.myapp.models.Student

// Add to classpath if in another location
// D:\> java -cp D:\javaapps com.myapp.models.Student

// #include (C++) vs import (Java)
// #include → Static Loading (at compile time)
// import   → Dynamic Loading (at runtime via JVM)
$ javac -d . Student.java && java com.myapp.Main
101 - Durga
💡java.lang is auto-imported — no import needed for String, System, Math, Thread, Object, etc. All other packages require explicit import statements.
