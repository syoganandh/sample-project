Wrapper Classes

Wrapper classes convert primitive data types into objects. Collections in Java can only store objects — not primitives — so wrapper classes bridge this gap. JDK 5.0 introduced Auto-Boxing and Auto-Unboxing to automate these conversions.

📦

Why Wrappers?

Collections only store objects. Wrapper classes convert primitives like int, float to Integer, Float objects.

🔁

Auto-Boxing

JDK 5.0 — automatic primitive → object conversion. No need for manual constructors.

📤

Auto-Unboxing

JDK 5.0 — automatic object → primitive conversion. Assign wrapper ref to primitive var.

🔒

Immutable

All wrapper class objects are immutable — once created, their value cannot change.
