IO Streams

A Stream is a channel that allows continuous data flow from input devices to a Java program and from a Java program to output devices. Java's java.io package provides all stream classes for file, console, and network I/O.

📥

InputStream

Byte-oriented. Reads data as bytes from source. Ends with InputStream.

📤

OutputStream

Byte-oriented. Writes data as bytes to destination. Ends with OutputStream.

📖

Reader

Character-oriented. Reads 2-byte Unicode chars. Ends with Reader.

✍️

Writer

Character-oriented. Writes Unicode chars to output. Ends with Writer.

import java.io.*;

// Method 1: FileWriter (character stream)
FileWriter fw = new FileWriter("output.txt");
fw.write("Hello Java IO!");
fw.close();

// Method 2: BufferedWriter (faster — buffered)
FileWriter   fw2 = new FileWriter("output.txt", true); // append
BufferedWriter bw = new BufferedWriter(fw2);
bw.write("Line 1");
bw.newLine();
bw.write("Line 2");
bw.flush();  // push buffer to file
bw.close();

// Method 3: PrintWriter (convenient)
PrintWriter pw = new PrintWriter("output.txt");
pw.println("Java PrintWriter");
pw.printf("Value: %d%n", 42);
pw.close();
import java.io.*;

// Method 1: FileReader + BufferedReader (line by line)
FileReader    fr = new FileReader("output.txt");
BufferedReader br = new BufferedReader(fr);

String line;
while((line = br.readLine()) != null) {
    System.out.println(line);
}
br.close();

// Method 2: try-with-resources (Java 7+ — auto close)
try (BufferedReader br2 = new BufferedReader(
        new FileReader("output.txt"))) {
    String l;
    while((l = br2.readLine()) != null)
        System.out.println(l);
} catch (IOException e) {
    e.printStackTrace();
}  // br2 auto-closed here

import java.io.*;

// Byte Streams — for binary data (images, audio, etc.)

// Write bytes to file
FileOutputStream fos = new FileOutputStream("data.bin");
BufferedOutputStream bos = new BufferedOutputStream(fos);
byte[] data = {65, 66, 67};    // A, B, C in ASCII
bos.write(data);
bos.close();

// Read bytes from file
FileInputStream  fis = new FileInputStream("data.bin");
BufferedInputStream bis = new BufferedInputStream(fis);
int b;
while((b = bis.read()) != -1) {
    System.out.print((char) b);   // A B C
}
bis.close();

// Serialization — save object state to file
import java.io.*;

class Student implements Serializable {
    int    sid   = 101;
    String sname = "Durga";
}

// Serialize (Object → File)
Student s = new Student();
ObjectOutputStream oos = new ObjectOutputStream(
    new FileOutputStream("student.ser"));
oos.writeObject(s);
oos.close();

// Deserialize (File → Object)
ObjectInputStream ois = new ObjectInputStream(
    new FileInputStream("student.ser"));
Student s2 = (Student) ois.readObject();
System.out.println(s2.sid + " " + s2.sname); // 101 Durga
ois.close();
101 Durga
💡Always close streams! Use try-with-resources (Java 7+) for automatic closing. Unclosed streams cause resource leaks and data corruption. Buffered streams improve performance by reducing disk access.
