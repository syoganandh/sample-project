Encapsulation

Bundling data and the methods that act on it inside one class — and hiding the internal details from the outside world.

The idea

Make fields private so they can't be touched directly. Expose controlled access through public getter and setter methods. The class decides what's allowed; the outside only sees a clean interface.

Example

public class BankAccount {
    private double balance;   // hidden field

    public double getBalance() {
        return balance;
    }

    public void deposit(double amount) {
        if (amount > 0) {     // guard the rules
            balance += amount;
        }
    }
}
Why the guard matters: outside code can't set a negative balance — it must go through deposit(), which enforces the rule.

Why use it

Control

Validate every change before it touches the data.

Safety

Internal state can't be set to invalid values.

Flexibility

Change the internals later without breaking callers.

Clarity

Users see a small, clear interface — not the mess.

One line to remember: keep fields private, expose behavior through methods.
