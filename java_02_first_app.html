Steps to Write First Java Application

Writing your first Java program requires installing the JDK, selecting an editor, writing the code, saving, compiling with javac, and running with java.

1

Install JDK

Download JDK from Oracle. Set PATH env variable to JDK/bin folder.

2

Select Editor

Use Notepad, Notepad++, EditPlus for learning. Use Eclipse/NetBeans in real-time.

3

Write Program

Write Java code using predefined API (Application Programming Interface).

4

Save File

Save as ClassName.java. If class is public, filename must match class name.

5

Compile

Run javac Test.java → generates Test.class (bytecode).

6

Run

Run java Test (no .class). JVM loads and executes the bytecode.

Program
Explanation
Output
// D:\javaapps\Test.java
class Test {
    public static void main(String[] args) {
        System.out.println("First Java Application");
    }
}
1
class Test — declares a class named Test. Every Java program must be inside a class.
2
public — JVM must access main() from outside, so it must be public.
3
static — JVM calls main() without creating an object, so it must be static.
4
void — main() doesn't return anything to JVM.
5
String[] args — command-line arguments passed at runtime (mandatory parameter).
6
System.out.println() — System is a class, out is a static PrintStream object, println() is a method.
Command Prompt:
D:\javaapps> javac Test.java
D:\javaapps> java Test

📁File Saving Rule: If the Java file contains a public class, the filename must exactly match the class name (case-sensitive). If no public class exists, any name is acceptable but matching the class name is recommended.
.java file
Source code file — written by the programmer, readable by humans
.class file
Bytecode file — generated by javac compiler, platform independent
JVM
Java Virtual Machine — executes bytecode on any OS (Windows, Linux, Mac)
javac
Java Compiler — converts .java → .class
java
Java Launcher — starts JVM and runs the .class bytecode
IDE
Eclipse, MyEclipse, NetBeans — preferred for real-time development
